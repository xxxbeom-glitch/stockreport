# AI 투자 경쟁앱 — 시스템 계약 (고정 기준)

**문서 성격:** 본 문서는 설명서가 아니라 **구현 변경의 위반 여부를 판단하는 절대 기준**이다.  
**적용 범위:** `src/trading/competition/` 신규 앱, REPLAY 검증, GitHub Pages 대시보드, Slack/주간·월간 리포트.  
**변경:** 본 문서의 규칙을 바꾸려면 변경 유형 **C (전략·성과 기준 변경)** 로 분류하고 **사용자 승인**이 있어야 한다.

## 1. 투자 대상 및 신규 진입 필터

| ID | 규칙 | 위반 시 |
|----|------|---------|
| U-01 | 신규 매수 대상은 **KOSPI·KOSDAQ 상장 일반 보통주**만 허용 | ETF·우선주 등으로 신규 매수 |
| U-02 | ETF, ETN, 우선주, SPAC, 상장 리츠는 **신규 매수 제외** | 해당 종목 신규 진입 |
| U-03 | 관리종목, 거래정지, 정리매매 등 **정상 매매·회수가 불안정한 고위험 종목**은 신규 매수 제외 | 위험상태 무시 신규 매수 |
| U-04 | 신규 종목 **최초 진입** 시 주문 시점 **현재가 100,000원 이하**만 허용 | 10만원 초과 신규 매수 |
| U-05 | 신규 진입 후보는 **최근 20거래일 평균 거래대금 30억원 이상**만 허용 | 유동성 미달 신규 매수 |
| U-06 | **이미 보유** 종목은 이후 가격이 100,000원을 초과해도 **추가매수** 판단 가능 | 보유 종목 추가매수 금지 |

**구현 참고:** `MAX_ENTRY_PRICE_KRW`, `MIN_AVG_TRADING_VALUE_KRW`, `evaluate_entry_eligibility()`, `classify_security_type()`.

## 4. 데이터·리플레이 공정성

| D-01 | 리플레이는 **해당 거래일 당시 알 수 있었던 데이터만** 사용 (**미래정보 금지**) | look-ahead |
| D-02 | 과거 리플레이의 종목군·가격·거래대금·위험·공시·뉴스·수급은 **해당 시점 기준**이어야 함 | 현재 시점 데이터로 과거 대체 |
| D-03 | 현재 종목 마스터/현재 시세를 과거 시점 **대체** 사용 시 → **정식 성과 검증 아님**, **개발 파이프라인 검증**으로만 표시 | 미표시 대체 사용 |
| D-04 | 팀 **핵심 전략 데이터 누락** 시 해당 팀 결과를 **정식 성과로 인정하지 않음** | 누락 상태 정식 인정 |
| D-05 | **C팀:** 기관·외국인 수급 데이터 없으면 **정식 수급 전략 평가 불가** | C팀 수급 없이 정식 평가 |
| D-06 | 데이터 누락을 임의 HOLD로 **위장하지 않음**; `data_invalid` 또는 **검증불가** 상태 기록 | 조용한 HOLD 대체 |

## 6. 정식 성과 vs 개발 검증

| 상태 | 의미 | 대시보드·리포트 표시 |
|------|------|---------------------|
| `formal_strategy_performance` | 본 계약 D·U 규칙을 **해당 campaign·거래일 기준으로 충족** 검증됨 | 정식 REPLAY 성과 |
| `development_validation_only` | 파이프라인·인프라 검증용; **미검증 항목이 남아 있음** | **개발 검증 데이터** (정식 성과 아님) |

**현재 canonical campaign `month_20260102_20260130_1b51cb`:**  
검증 완료 전까지 **`development_validation_only`** (상세는 `validation_status.json` 및 코드 `validation_contract.py`).
