# AI 투자 경쟁앱 — 변경 가드 (필수 보고 형식)

모든 **코드·워크플로·데이터 경로** 변경 완료 후, PR/커밋 설명 또는 Cursor 완료 보고에 **아래 항목을 빠짐없이** 포함한다.  
위반 여부 판단 기준: [`SYSTEM_CONTRACT.md`](SYSTEM_CONTRACT.md).

---

## 필수 보고 템플릿

```markdown
## 변경 보고

- **변경 목적:**
- **변경 유형:** A 실행 안정화 / B 데이터 공급 변경 / C 전략·성과 기준 변경
- **수정 파일:**
- **`SYSTEM_CONTRACT` 위반 여부:** 없음 / 있음 (해당 조항 ID: U-__, T-__, …)
- **투자 판단 결과에 영향을 주는지:** 예 / 아니오
- **리플레이 공정성에 영향을 주는지:** 예 / 아니오
- **누락되거나 대체된 데이터:**
- **임시 우회 여부:** 없음 / 있음 (사유·제거 조건)
- **정식 성과 사용 가능 여부:** 가능 / 불가 (`development_validation_only` 등)
- **테스트 결과:**
- **commit / push:**
```

## 판정 원칙

### 변경 유형

| 유형 | 허용 조건 |
|------|-----------|
| **A** 실행 안정화 | `SYSTEM_CONTRACT` **위반 없음**. 인증·캐시·워크플로 UX·observability·버그 수정 |
| **B** 데이터 공급 변경 | **계약 영향 분석** 후 적용. 시점·소스·필터 입력이 D-02·D-03·U-01~U-06과 충돌하지 않는지 명시 |
| **C** 전략·성과 기준 변경 | **사용자 명시 승인** 후에만 구현. `SYSTEM_CONTRACT.md` 개정 필요 |

### 정식 성과

- `정식 성과 사용 가능 여부: 불가` 이면 대시보드·Slack·리포트에 **「개발 검증 데이터」** 표시 필수.
- C팀 수급·과거 시점 마스터·위험상태 시점 등 **미검증 항목**이 남으면 `development_validation_only` 유지.

### 금지 사례 (예시)

- 데이터 누락을 전 팀 HOLD로 위장 → **D-06 위반**
- 현재 `static_ticker_master`로 2026-01-09 리플레이를 **정식 성과**로 표시만 변경 → **D-02·D-03 위반** (검증 없이)
- 1200종목 cap 등 **임의 후보 축소**를 전체 스캔 규칙 대체 → **U-01~U-05 위반**
- KIS/pykrx 소스를 사용자 승인 없이 바꾸며 B 유형 미기재 → **프로세스 위반**

## 에이전트·개발자 의무

1. 수정 **전** 변경 유형(A/B/C)을 스스로 분류한다.
2. B·C이면 `SYSTEM_CONTRACT` 해당 조항을 인용한다.
3. REPLAY campaign 상태(`validation_contract.py`)와 모순되지 않는지 확인한다.
4. 본 템플릿 없는 「수정 완료」 보고는 **불완전**으로 간주한다.
