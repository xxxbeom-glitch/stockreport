# 데이터 파이프라인 설계: 국장/미장

국장과 미장은 같은 시스템 구조를 사용하되, 데이터 소스와 판단 가중치를 다르게 둔다.

## 공통 처리 흐름

```txt
Raw 데이터 수집
→ 정규화
→ 시장/섹터/종목 단위 분류
→ AI 분석 context 생성
→ AI 투표 입력 생성
→ UI 출력 데이터 생성
```

## 국장 KR 파이프라인

국장은 수급, 테마, 뉴스 속도가 중요하다.

### 주요 데이터 소스

- 한국투자증권 파이프라인/API
- 현재가
- 등락률
- 거래량
- 목표가
- 52주 고가/저가
- 외국인 수급
- 기관 수급
- 개인 수급
- 코스피
- 코스닥
- 원/달러 환율
- 금리
- 미국장 주요 지수
- 선물 흐름
- 종목 뉴스
- 기업 공시
- 실적 발표
- 정책 이슈
- 산업 이슈
- 증권사 리포트 요약
- 강한 섹터
- 약한 섹터
- 자금 유입 섹터
- 자금 이탈 섹터
- 관련 종목
- 테마 확산 여부
- X/커뮤니티/텔레그램 반응

### 국장 판단 가중치

```txt
수급 > 테마 > 뉴스/공시 > 시장지표 > 커뮤니티 반응
```

## 미장 US 파이프라인

미장은 실적, 가이던스, 매크로, 기관 흐름이 중요하다.

### 주요 데이터 소스

- 현재가
- 등락률
- 거래량
- 시가총액
- EPS
- PER
- 매출/영업이익
- 실적 발표
- 가이던스
- 애널리스트 목표가
- 애널리스트 투자의견
- 나스닥
- S&P500
- 다우
- 미국 국채금리
- 달러 인덱스
- CPI/PCE/FOMC 일정
- ETF 자금 흐름
- 옵션 거래 흐름
- 내부자 거래
- SEC 공시
- X/Reddit/Stocktwits 반응

### 미장 판단 가중치

```txt
실적/가이던스 > 매크로 > 기관/ETF 흐름 > 뉴스 > 커뮤니티 반응
```

## 구현 기준

```js
marketType: "KR" | "US"
```

`marketType` 값에 따라 데이터 소스와 판단 가중치를 다르게 적용한다.
