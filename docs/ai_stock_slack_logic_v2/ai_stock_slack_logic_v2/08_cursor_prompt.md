# 08. Cursor 구현 프롬프트

아래 내용을 Cursor에 붙여넣어 구현한다.

```text
AI 자동 스캔 + 슬랙 메시지 발송 로직을 구현해줘.

목표:
- 전체 시장 종목을 추천하지 않는다.
- 기존에 등록한 관심종목 25개만 대상으로 스캔한다.
- AI가 장중 데이터를 보고 “1주 테스트 진입을 검토할 만한 종목”만 슬랙으로 보낸다.
- 비추천, 진입 보류, 주의 필요, 판단 애매한 종목은 슬랙 메시지를 보내지 않는다.
- 앱 리포트에는 전체 결과를 보여줘도 되지만, 슬랙은 실제로 볼 만한 종목만 보내는 알림으로 제한한다.

관심종목 기준:
- 반도체 소재
- 반도체 부품
- 반도체 장비
- 방산·우주
- 조선·기자재

각 섹터 안의 종목은 기존에 등록한 25개 관심종목 데이터만 사용한다.
해운 섹터나 기타 종목은 포함하지 않는다.

분석 목적:
- 지금 이 종목이 새로 진입해볼 만한 위치인지 판단한다.
- 추격매수는 절대 권장하지 않는다.
- 급등한 종목은 바로 매수 추천하지 않고, 눌림 구간이 있는지 판단한다.
- 가능하면 “예약가 후보”를 범위로 제안한다.
- 사용자는 1주 단위로 테스트 진입할 예정이므로 메시지는 소액 테스트 관점으로 작성한다.

분석 시간대:
1. 09:30
   - 장초반 변동성이 어느 정도 줄어든 뒤 1차 스캔
   - 거래대금이 붙는 종목 확인
   - 테스트 진입 검토 또는 예약가 후보 제안

2. 10:50
   - 오전 흐름 유지 여부 확인
   - 기존 예약가가 아직 유효한지 판단
   - 유효하면 유지, 아니면 메시지 발송하지 않음

3. 13:50
   - 오후에 돈이 다시 들어오는 종목 확인
   - 오전보다 흐름이 좋아진 종목만 메시지 생성

4. 14:50
   - 마감 전 수급 확인
   - 당일 신규 진입보다는 내일 관찰 강화 후보 판단
   - 그래도 1주 테스트 진입이 가능해 보이는 종목만 메시지 발송

AI가 확인해야 할 데이터:
- 현재가
- 전일 종가
- 당일 고가/저가
- 최근 고점 대비 조정률
- 최근 지지 구간
- 거래대금 변화
- 외국인 순매수
- 기관 순매수
- 같은 섹터 내 다른 종목들의 동반 움직임
- 관련 뉴스/이슈
- 급등 추격 구간 여부
- 눌림 진입 가능 여부

슬랙 발송 허용 상태:
- 테스트 진입 검토
- 예약가 제안
- 관찰 강화
- 눌림 진입 가능
- 수급 반전 감지

슬랙 발송 금지 상태:
- 비추천
- 진입 보류
- 주의 필요
- 추격매수 위험
- 거래대금 부족
- 수급 약함
- 판단 애매
- 데이터 부족

중요 로직:
1. 관심종목 25개를 시간대별로 스캔한다.
2. 각 종목별로 AI 판단 상태를 만든다.
3. 발송 허용 상태에 해당하는 종목만 슬랙 메시지를 생성한다.
4. 발송 금지 상태인 종목은 슬랙 메시지를 만들지 않는다.
5. 같은 종목은 하루에 여러 번 반복 발송하지 않는다.
6. 단, 예약가가 크게 바뀌거나 오전보다 오후 흐름이 확실히 좋아진 경우에는 다시 발송할 수 있다.
7. 메시지는 최대 1~3개 종목만 보내도록 제한한다.
8. 조건에 맞는 종목이 없으면 슬랙 메시지를 보내지 않는다.

AI 에이전트 구조:

1. MarketDataAgent
- 관심종목 25개의 현재가, 전일 종가, 당일 고가/저가, 거래대금, 외국인/기관 수급, 관련 뉴스 데이터를 수집한다.
- 전체 시장 종목은 수집하지 않는다.

2. SectorMoodAgent
- 반도체 소재, 반도체 부품, 반도체 장비, 방산·우주, 조선·기자재 섹터의 분위기를 판단한다.
- 섹터 분위기를 strong / neutral / weak 로 분류한다.

3. WatchlistPickAgent
- 관심종목 25개 중 오늘 관찰 강도를 높일 종목만 고른다.
- 거래대금 증가, 수급 개선, 섹터 동반 상승, 뉴스 모멘텀이 있는 종목을 우선한다.

4. EntryPriceAgent
- 선택된 종목에 대해 신규 진입 검토가 가능한지 판단한다.
- 추격매수 구간이면 제외한다.
- 눌림 진입이 가능해 보이면 예약가 후보를 범위로 계산한다.

5. SlackMessageAgent
- 발송 대상 종목에 대해서만 슬랙 메시지를 만든다.
- 메시지는 사용자가 1주 테스트 진입을 검토할 수 있게 직관적으로 작성한다.

6. SendFilterAgent
- 최종 발송 여부를 결정한다.
- 발송 금지 상태는 절대 슬랙으로 보내지 않는다.

슬랙 메시지 포맷:

[테스트 진입 검토] 종목명

현재가:
예약가 후보:

판단:
왜 지금 볼 만한지 2~3문장으로 설명한다.

진입 관점:
추격매수가 아니라, 눌림이 나오면 1주 테스트 진입을 검토할 만하다는 식으로 작성한다.

주의 조건:
어떤 조건이 깨지면 진입을 취소해야 하는지 작성한다.

표현 규칙:
- “무조건 매수하세요” 금지.
- “이 가격에 사세요” 금지.
- “예약가 후보”, “진입 검토”, “1주 테스트”, “분할 진입 관점” 표현 사용.
- 비추천 종목은 슬랙으로 보내지 않음.
- 부정적 판단은 내부 로그 또는 앱 리포트에만 남김.

우선 실제 증권 API 연동 전에는 더미 데이터로 다음 구조를 먼저 구현해줘:
- 관심종목 스캔 함수
- 시간대별 실행 함수
- AI 판단 상태 생성 함수
- 슬랙 발송 여부 필터 함수
- 슬랙 메시지 생성 함수
- 발송 로그 저장 구조
```
