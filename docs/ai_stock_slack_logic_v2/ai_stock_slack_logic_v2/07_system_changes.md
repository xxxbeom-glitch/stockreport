# 07. 관련 시스템 수정 요구사항

## 1. 관심종목 DB 고정

현재 시스템은 전체 시장을 스캔하는 것이 아니라 관심종목 25개만 대상으로 해야 한다.

필요 작업:

- 관심 섹터/종목 JSON 또는 DB 파일 생성
- 종목명, 섹터, 주요사업 저장
- 해운 섹터 제외
- 기타 종목 자동 추천 금지

## 2. 앱 리포트 화면 수정

앱 화면은 다음 구조를 유지한다.

```text
시장 주요 지표
→ 관심 섹터 흐름
→ 관찰이 필요한 종목
```

관찰이 필요한 종목 카드에는 다음 정보를 사용한다.

- 종목명
- 현재가
- 목표가
- 외국인 순매수
- 52주 최고가
- 주요사업
- 선정이유

## 3. 자동 스캔 스케줄러 추가

필요 실행 시간:

- 09:30
- 10:50
- 13:50
- 14:50

각 시간마다 관심종목 25개를 스캔한다.

## 4. AI 판단 상태 생성

각 종목별로 판단 상태를 만든다.

발송 허용 상태:

- 테스트 진입 검토
- 예약가 제안
- 관찰 강화
- 눌림 진입 가능
- 수급 반전 감지

발송 금지 상태:

- 비추천
- 진입 보류
- 주의 필요
- 추격매수 위험
- 거래대금 부족
- 수급 약함
- 판단 애매
- 데이터 부족

## 5. 슬랙 발송 필터 추가

슬랙은 발송 허용 상태만 보낸다.

부정적 판단은 다음 위치에만 저장한다.

- 앱 리포트
- 내부 로그
- 분석 히스토리

## 6. 발송 로그 저장

반복 발송을 막기 위해 로그를 저장한다.

저장 항목:

- 날짜
- 실행 시간대
- 종목명
- 판단 상태
- 현재가
- 예약가 후보
- 발송 여부
- 발송 메시지
- 발송하지 않은 이유

## 7. 더미 데이터 우선 구현

실제 증권 API 연동 전에는 더미 데이터로 다음 흐름을 먼저 구현한다.

- 관심종목 스캔 함수
- 시간대별 실행 함수
- AI 판단 상태 생성 함수
- 슬랙 발송 여부 필터 함수
- 슬랙 메시지 생성 함수
- 발송 로그 저장 구조
