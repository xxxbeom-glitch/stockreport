# 03. 데이터 수집 및 선별 로직

## 기본 흐름

```text
관심종목 25개 데이터 수집
→ 섹터 분위기 판단
→ 종목별 현재 위치 판단
→ 진입 검토 가능 종목 선별
→ 예약가 후보 계산
→ 발송 가능 상태 필터링
→ 슬랙 메시지 생성
```

## 수집해야 할 데이터

종목별로 다음 데이터를 수집한다.

- 현재가
- 전일 종가
- 당일 고가/저가
- 최근 고점 대비 조정률
- 최근 지지 구간
- 거래대금 변화
- 최근 3개월 평균 거래대금 대비 현재 거래대금
- 외국인 순매수
- 기관 순매수
- 같은 섹터 내 종목들의 동반 움직임
- 관련 뉴스/이슈
- 목표가 또는 참고 가격
- 52주 최고가
- 주요사업

## 선별 기준

슬랙 발송 후보는 다음 조건 중 여러 개를 만족해야 한다.

- 거래대금이 평소보다 증가
- 외국인/기관 수급 개선
- 같은 섹터 내 종목들이 함께 움직임
- 관련 뉴스나 이슈가 발생
- 현재가가 당일 고점 추격 구간이 아님
- 눌림 진입 후보 가격대를 계산할 수 있음
- 최근 고점 대비 부담이 크지 않음

## 제외 기준

다음 경우는 슬랙 메시지를 보내지 않는다.

- 거래대금 부족
- 수급 약함
- 뉴스는 있지만 실제 매수세 없음
- 당일 급등 후 고점 추격 위험
- 판단 애매
- 데이터 부족
- 비추천
- 진입 보류
- 주의 필요

## 예약가 후보 산정 기준

예약가 후보는 단일 가격이 아니라 범위로 제안한다.

예:

```text
현재가: 32,400원
예약가 후보: 31,600원 ~ 31,900원
```

예약가 후보 계산 시 참고한다.

- 현재가 대비 과도하게 높지 않은 구간
- 당일 저가 부근
- 전일 종가 부근
- 최근 지지 구간
- 급등 후 눌림 가능 구간
- 거래대금이 유지되는 조건

## 발송 개수 제한

- 한 번의 스캔에서 최대 1~3개 종목만 슬랙 발송
- 조건에 맞는 종목이 없으면 발송하지 않음
- 같은 종목은 하루에 반복 발송하지 않음
- 단, 예약가가 크게 바뀌거나 오전보다 오후 흐름이 확실히 좋아진 경우 재발송 가능
