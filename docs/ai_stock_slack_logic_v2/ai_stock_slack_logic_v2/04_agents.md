# 04. AI 에이전트별 임무

## 전체 구조

이 시스템은 하나의 AI가 모든 판단을 한 번에 처리하지 않고, 역할별 에이전트로 나눈다.

```text
MarketDataAgent
→ SectorMoodAgent
→ WatchlistPickAgent
→ EntryPriceAgent
→ SlackMessageAgent
→ SendFilterAgent
```

## 1. MarketDataAgent

### 역할

관심종목 25개의 장중 데이터를 수집한다.

### 수집 대상

- 현재가
- 전일 종가
- 당일 고가/저가
- 거래대금
- 외국인/기관 수급
- 관련 뉴스/이슈
- 52주 최고가
- 목표가 또는 참고 가격

### 규칙

- 전체 시장 종목은 수집하지 않는다.
- 해운 섹터는 제외한다.
- 관심종목 DB에 없는 종목은 분석하지 않는다.

## 2. SectorMoodAgent

### 역할

관심 섹터별 분위기를 판단한다.

### 판단 대상

- 반도체 소재
- 반도체 부품
- 반도체 장비
- 방산·우주
- 조선·기자재

### 결과 예시

```text
반도체 장비: strong
반도체 소재: neutral
조선·기자재: weak
```

## 3. WatchlistPickAgent

### 역할

25개 관심종목 중 오늘 볼 만한 후보만 1차 선별한다.

### 선별 기준

- 거래대금 증가
- 외국인/기관 수급 개선
- 섹터 동반 상승
- 뉴스 모멘텀
- 급등 추격이 아닌 위치

## 4. EntryPriceAgent

### 역할

선별된 종목에 대해 1주 테스트 진입이 가능한지 판단하고 예약가 후보를 계산한다.

### 판단 기준

- 현재가가 당일 고점 근처인지
- 최근 고점 대비 조정률
- 당일 저점
- 전일 종가
- 최근 지지선
- 거래대금 유지 여부
- 수급 개선 여부

### 결과 예시

```json
{
  "status": "예약가 제안",
  "currentPrice": "32,400원",
  "entryRange": "31,600원 ~ 31,900원",
  "isChasing": false
}
```

## 5. SlackMessageAgent

### 역할

발송 대상 종목에 대해서만 슬랙 메시지를 작성한다.

### 메시지 톤

- 직관적
- 짧게
- 1주 테스트 관점
- 추격매수 방지
- 예약가 후보 중심

## 6. SendFilterAgent

### 역할

최종 발송 여부를 결정한다.

### 발송 허용 상태

- 테스트 진입 검토
- 예약가 제안
- 관찰 강화
- 눌림 진입 가능
- 수급 반전 감지

### 발송 금지 상태

- 비추천
- 진입 보류
- 주의 필요
- 추격매수 위험
- 거래대금 부족
- 수급 약함
- 판단 애매
- 데이터 부족

### 최종 규칙

- 발송 금지 상태는 슬랙 메시지를 만들지 않는다.
- 부정적 판단은 앱 리포트나 내부 로그에만 남긴다.
- 조건에 맞는 종목이 없으면 슬랙을 보내지 않는다.
