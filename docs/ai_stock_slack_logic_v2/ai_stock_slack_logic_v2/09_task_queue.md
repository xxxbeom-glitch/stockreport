# 09. Cursor 순차 작업 체크리스트

## 목적

컨텍스트 한도를 피하기 위해 Cursor가 한 번에 모든 문서를 처리하지 않고, 한 번에 하나의 문서만 처리하도록 만든다.

## 작업 규칙

- 아직 체크되지 않은 첫 번째 항목만 처리한다.
- 처리한 뒤 체크박스를 완료 처리한다.
- `10_progress.md`에 수정 파일, 작업 내용, 다음 작업을 기록한다.
- 다음 파일은 건드리지 않고 멈춘다.
- 컨텍스트가 길어질 것 같으면 즉시 멈추고 `10_progress.md`에 현재 상태를 남긴다.

## Cursor 실행 프롬프트

```text
09_task_queue.md와 10_progress.md를 확인해서 아직 처리하지 않은 첫 번째 md 파일 1개만 반영해줘.

규칙:
- 한 번에 모든 파일을 처리하지 말 것.
- 아직 체크되지 않은 첫 번째 항목만 처리할 것.
- 완료 후 09_task_queue.md 체크박스를 완료 처리할 것.
- 10_progress.md에 작업 내용, 수정 파일, 다음 단계 기록을 남길 것.
- 다음 파일은 건드리지 않고 멈출 것.
- 컨텍스트가 길어질 것 같으면 즉시 멈추고 10_progress.md에 현재 상태를 기록할 것.

지금은 첫 번째 미완료 항목부터 진행해줘.
```
